# Звіт з виконання лабораторної роботи №2

**Тема:** Побудова СІ-конвеєра із використанням GitHub Actions

## 1. Що було зроблено

Під час виконання лабораторної роботи я налаштувала автоматичне виконання коду (CI-конвеєр) для свого проєкту. 

**Основні етапи:**
* Створено файл конвеєра `ci.yml`, який автоматично запускається на серверах GitHub (`ubuntu-latest`).
* Застосовано матричну стратегію (matrix), щоб паралельно тестувати 4 модулі проєкту: завантаження, очищення, дослідження та візуалізацію.
* Налаштовано автоматичне збереження результатів (артефактів) — тепер графіки та CSV-файли можна завантажити прямо з GitHub після успішного виконання коду.
* Підключено власний комп'ютер як локальний сервер (self-hosted runner).

---

## 2. Порівняння GitHub-hosted та Self-hosted runner

Відповідно до завдання, я порівняла роботу хмарного агента від GitHub та локального агента на своєму ПК.

### 2.1. Швидкість виконання
* **GitHub-hosted:** Працює трохи довше, оскільки щоразу створює нову віртуальну машину, "з нуля" завантажує код, встановлює Python та всі бібліотеки.
* **Self-hosted (локальний):** Запускається швидше, адже Python та всі необхідні бібліотеки (pandas, matplotlib тощо) вже встановлені на моєму комп'ютері.

### 2.2. Доступ до локальних ресурсів
* **GitHub-hosted:** Не має доступу до мого комп'ютера, тому змушений щоразу завантажувати набори даних з інтернету.
* **Self-hosted:** Має прямий доступ до мого жорсткого диска. Це дуже зручно для роботи з великими датасетами, які вже лежать у папці проєкту, адже їх не треба нізвідки качати.

### 2.3. Ризики
* **Безпека:** Локальний агент небезпечніший, бо він має доступ до моїх особистих файлів. Хмарний агент безпечний, бо після виконання коду віртуальна машина просто видаляється.
* **Стабільність:** Мій локальний конвеєр буде працювати лише тоді, коли комп'ютер увімкнений і відкритий термінал (runner може бути offline). Хмарні сервери GitHub працюють завжди.

У цій лабораторній роботі я навчилася використовувати GitHub Actions для автоматизації процесів. Я успішно налаштувала хмарний та локальний CI-конвеєри та навчилася зберігати артефакти виконання.
